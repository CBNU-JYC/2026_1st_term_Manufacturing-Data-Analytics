# clean_dataset.py 설명 보고서

## 1. 파일 개요

- 원본 코드 파일: [clean_dataset.py](/Users/jeong-yongcheol/Desktop/00_CBNU_AI/My_project/ManDA_Lecture/3rd_project_0317/clean_dataset.py)

- 입력 데이터 파일: `labeled_data.csv`
- 출력 데이터 파일: `labeled_data_clean.csv`

이 코드는 원본 제조 데이터를 불러온 뒤, 데이터 품질 관점에서 문제를 점검하고 정제하여 더 신뢰할 수 있는 데이터셋으로 저장하는 전처리 스크립트이다.

## 2. 코드의 목적

- 중복 데이터 제거
- 핵심 센서값 결측치 제거
- 물리적으로 불가능한 시간 데이터 제거
- 논리적으로 모순되는 데이터 제거
- 비정상적인 온도 데이터 제거
- 최종적으로 정제된 고품질 데이터셋 생성

즉, 이 코드는 머신러닝 분석이나 통계 분석을 수행하기 전에 데이터 품질을 먼저 확보하는 것을 목적으로 한다.

## 3. 사용된 데이터 품질 기준

코드는 다음 5가지 기준으로 데이터를 검사한다.

### 3.1 유일성 (Uniqueness)

- 같은 기록이 중복으로 저장되어 있는지 확인한다.
- 기준 컬럼은 `TimeStamp`, `EQUIP_CD`, `PART_NAME`이다.
- 이 세 값이 모두 같으면 같은 생산 기록으로 판단한다.

### 3.2 완전성 (Completeness)

- 핵심 정보가 비어 있지 않은지 확인한다.
- 핵심 컬럼은 `Injection_Time`, `Cycle_Time`, `Barrel_Temperature_1`이다.
- 이 중 하나라도 결측치가 있으면 해당 행을 제거한다.

### 3.3 유효성 (Validity)

- 값이 물리적으로 가능한 범위인지 확인한다.
- 사출 시간과 사이클 시간은 0보다 커야 한다.
- 0 이하의 값은 잘못된 데이터로 판단한다.

### 3.4 일관성 (Consistency)

- 서로 관련된 값들 사이에 논리적 충돌이 없는지 확인한다.
- `PassOrFail`이 `Y`인데 `Reason`이 존재하면 모순으로 판단한다.
- `Injection_Time`이 `Cycle_Time`보다 크거나 같으면 모순으로 판단한다.

### 3.5 정확성 (Accuracy)

- 측정값이 실제 공정 상식에 맞는지 확인한다.
- `Barrel_Temperature_1`이 50 미만 또는 400 초과이면 비정상 데이터로 간주한다.

## 4. 코드 동작 과정

### 4.1 데이터 불러오기

- `pandas.read_csv()`를 사용해 `labeled_data.csv`를 읽는다.
- 전체 행 수를 출력해 원본 데이터 규모를 확인한다.

### 4.2 복사본 생성

- `df.copy()`를 사용해 원본 데이터 복사본 `df_clean`을 만든다.
- 원본은 유지하고 정제는 복사본에서 진행한다.

### 4.3 유일성 검사

- `duplicated()`를 이용해 중복 데이터를 찾는다.
- `drop_duplicates()`를 이용해 첫 번째 기록만 남기고 제거한다.

### 4.4 완전성 검사

- `isnull()`과 `dropna()`를 이용해 핵심 센서값 결측 데이터를 제거한다.

### 4.5 유효성 검사

- `Cycle_Time <= 0` 또는 `Injection_Time <= 0`인 행을 제거한다.

### 4.6 일관성 검사

- 양품인데 불량 사유가 있는 경우 제거한다.
- 사출 시간이 전체 사이클 시간보다 긴 경우 제거한다.

### 4.7 정확성 검사

- 온도 범위를 벗어난 행을 제거한다.

### 4.8 결과 저장

- 정제 전후 행 수를 출력한다.
- 최종 데이터를 `labeled_data_clean.csv`로 저장한다.

## 5. 이 코드를 통해 학습할 수 있는 내용

### 5.1 데이터 전처리의 중요성

- 원본 데이터는 그대로 분석에 쓰기 어렵다는 점을 배울 수 있다.
- 데이터 품질이 낮으면 모델 성능과 분석 결과도 신뢰하기 어렵다.

### 5.2 pandas 기본 활용

- CSV 읽기와 저장
- 복사본 생성
- 조건 필터링
- 결측치 검사
- 중복 제거

### 5.3 데이터 품질 관리 개념

- 유일성
- 완전성
- 유효성
- 일관성
- 정확성

이 다섯 개념은 실무 데이터 전처리에서도 자주 사용되는 핵심 기준이다.

### 5.4 도메인 지식의 필요성

- 어떤 값이 비정상인지 판단하려면 공정 지식이 필요하다.
- 예를 들어 온도 범위나 시간 관계는 현장 지식을 반영한 규칙이다.

### 5.5 정제 전략의 차이

- 결측치는 제거할 수도 있고 대체할 수도 있다.
- 이상치는 제거할 수도 있고 치환할 수도 있다.
- 이 코드는 가장 이해하기 쉬운 제거 중심 전략을 사용한다.
